# 普通微信（个人微信）接入分析

> 分析如何像 Hermes Agent 那样通过扫码登录普通微信，调用微信 API 收发消息。
> 回答核心问题：**难不难？怎么设计？有什么风险？**

---

## 一、先说结论：难不难？

| 维度 | 评估 |
|------|------|
| **技术难度** | ⭐⭐ 中等（有现成的开源库，几十行代码就能跑起来） |
| **稳定性** | ⭐⭐⭐⭐⭐ **极差**（随时可能被封号、协议失效） |
| **合规性** | ❌ 违反微信使用条款（ToS） |
| **生产可用性** | ❌ 不适合生产环境（除非愿意承担封号风险） |
| **开发调试体验** | ⭐⭐ 繁琐（每次重启都要重新扫码） |

**一句话**：技术上不难，但**微信会封号**。个人微信号一旦被检测到自动化操作，轻则限制登录，重则永久封禁。这不是危言耸听——itchat-uos 的 README 里明确写了"大多数使用者可能出现微信被官方封禁提醒"。

---

## 二、Hermes Agent 是怎么接入微信的？

从 Hermes Agent 的 README 中看到：

> 🔌 **HermesClaw** — Community WeChat bridge: Run Hermes Agent and OpenClaw on the same WeChat account.

Hermes 官方仓库里的 WeChat 接入是通过社区插件 **HermesClaw** 实现的（一个独立项目）。它的设计思路是：

```
微信桌面客户端（Windows）
    │
    ▼  UI Automation（UIAutomation）
    │  模拟人类操作：点击、读取消息、发送消息
    │
HermesClaw Bridge
    │
    ▼  HTTP / WebSocket
    │
Hermes Agent
```

**核心原理**：不是破解微信协议，而是**像人一样操控微信客户端**。微信客户端安装在 Windows 上，程序通过 Windows UI Automation 接口读取消息列表、提取文字、发送回复。

这就是 **wxauto** 库做的事情。HermesClaw 大概率也是基于类似的技术。

## 三、三种技术方案对比

### 方案 A：Web 协议（itchat / itchat-uos）

```
┌─────────────┐     网页版微信协议      ┌─────────────┐
│  Python 程序  │ ◄───────────────────► │  微信服务器   │
└─────────────┘    (HTTP + WebSocket)    └─────────────┘
      │
      ▼  扫码登录
┌─────────────┐
│  手机微信扫码  │
└─────────────┘
```

| 项目 | 信息 |
|------|------|
| **代表库** | `itchat`（已死）、`itchat-uos`（半死不活） |
| **GitHub Star** | itchat: 25k（历史），itchat-uos: 747 |
| **稳定性** | ⚠️ 微信从 2017 年起逐步封杀网页版，新号基本登不上 |
| **封号风险** | 🔴🔴🔴 **极高**，作者自己在 README 写"多数使用者被封" |
| **最后更新** | 2023 年，已停更 |
| **是否推荐** | ❌ 不推荐 |

**代码示例**（跑不起来的概率很大）：

```python
import itchat

# 扫码登录
itchat.auto_login()

# 收到消息自动回复
@itchat.msg_register(itchat.content.TEXT)
def reply(msg):
    # 调用调度引擎
    result = workflow.run(msg.text)
    return format_result(result)

itchat.run()
```

### 方案 B：UI Automation（wxauto）

```
┌──────────────────┐
│  Windows 电脑     │
│                  │
│  ┌─────────────┐ │
│  │ 微信桌面客户端 │ │  ← 必须打开，保持登录状态
│  │ (真实微信)    │ │
│  └──────┬──────┘ │
│         │         │
│  ┌──────▼──────┐ │    UI Automation（读界面元素）
│  │  wxauto     │ │    模拟鼠标键盘操作
│  │  Python 库   │ │
│  └──────┬──────┘ │
│         │         │
│  ┌──────▼──────┐ │
│  │ 调度系统     │ │
│  │ Workflow    │ │
│  └─────────────┘ │
└──────────────────┘
```

| 项目 | 信息 |
|------|------|
| **代表库** | `wxauto`（免费版）、`wxauto Plus`（付费版） |
| **GitHub Star** | 7.2k |
| **原理** | Windows UIAutomation，操控真实的微信桌面客户端 |
| **稳定性** | 🟡 中等（依赖微信版本更新，库需要跟进适配） |
| **封号风险** | 🟡 较低（操作的是真实客户端，不是破解协议） |
| **限制** | **必须运行在 Windows 上**，微信客户端必须保持登录 |
| **最后更新** | 2025 年（仍在维护） |
| **是否推荐** | ⚠️ 可以尝试（个人项目），生产风险仍存在 |

**代码示例**：

```python
from wxauto import WeChat

wx = WeChat()  # 自动获取当前已登录的微信窗口

# 监听消息
def listen():
    while True:
        msgs = wx.GetNextNewMessage()
        if msgs:
            for msg in msgs:
                user = msg[0]   # 发送者
                text = msg[1]   # 消息内容
                
                # 调用调度引擎
                result = workflow.run(text)
                reply = format_result(result)
                
                # 回复
                wx.SendMsg(reply, user)
```

**优点**：
- 操作真实客户端，比破解协议安全
- 不需要破解密钥、不需要逆向工程
- API 简洁（发消息、读消息、获取联系人）

**缺点**：
- 必须 Windows，不支持 macOS 服务器
- 微信客户端占用资源，不适合长时间无人值守
- 微信版本升级可能导致库失效
- 扫码登录不能持久（每次重启客户端需要重新扫码）

## 五、如果你想尝试：推荐 wxauto（方案 B）

### 为什么推荐它？

1. **Hermes Agent 走的也是这条路** — HermesClaw 就是基于 UI Automation
2. **不破解协议** — 操作真实微信客户端，封号风险最低
3. **API 足够简单** — 几行代码就能收发消息
4. **免费** — 不需要买 Token

### 前置条件

- 一台 **Windows 电脑**（或 Windows 虚拟机）
- 安装**微信桌面客户端**（保持登录）
- Python 3.8+

### 需要新增的依赖

```txt
wxauto>=4.0
```

### 架构设计

```python
# base.py — 统一通道接口
class Channel(ABC):
    name: str
    
    @abstractmethod
    async def start(self): ...
    
    @abstractmethod
    async def send_message(self, user_id: str, content: str): ...
    
    @abstractmethod
    async def handle_message(self, user_id: str, text: str) -> str: ...


# handler.py — 微信通道实现
import threading
from wxauto import WeChat
from app.orchestration.workflow import Workflow

class WeChatHandler(Channel):
    """个人微信消息通道
    
    运行在独立线程中，持续监听微信消息。
    调用 Workflow.run() 处理调度指令，通过微信回复结果。
    """
    
    name = "wechat"
    
    def __init__(self, config: dict, workflow: Workflow):
        self.wx = WeChat()
        self.workflow = workflow
        self.allowed_users = set(config.get("allowed_users", []))
        self._running = False
        self._thread = None
    
    def start(self):
        """启动消息监听（在后台线程中运行）"""
        self._running = True
        self._thread = threading.Thread(target=self._listen_loop, daemon=True)
        self._thread.start()
        print("🤖 微信通道已启动，等待消息...")
    
    def _listen_loop(self):
        """消息监听循环"""
        while self._running:
            try:
                msgs = self.wx.GetNextNewMessage()
                if msgs:
                    for user, text in msgs:
                        if user in self.allowed_users or not self.allowed_users:
                            reply = self.process_instruction(text)
                            self.wx.SendMsg(reply, user)
            except Exception as e:
                print(f"微信消息处理异常: {e}")
            time.sleep(1)
    
    def process_instruction(self, text: str) -> str:
        """处理一条指令"""
        if text.strip() in ("帮助", "help", "?"):
            return HELP_TEXT
        try:
            state = self.workflow.run(text)
            return format_result(state)
        except Exception as e:
            return f"❌ 调度失败: {e}"
    
    def stop(self):
        self._running = False
```

### server.py 改动

```python
# 在启动事件中初始化微信通道
@app.on_event("startup")
async def startup():
    wechat_config = load_config("wechat_config.json")
    if wechat_config and wechat_config.get("enabled"):
        handler = WeChatHandler(wechat_config, get_workflow())
        handler.start()
```

**非常少**的改动量（约 15 行）。

## 七、最终建议

### 如果只是个人项目、学习探索

可以尝试 **wxauto**（方案 B），体验一下"扫码登录 → 自动回复"的流程。代码量不大，几小时就能跑起来。

### 如果是严肃的仓库调度系统（生产环境）

**强烈建议回到企业微信自建应用方案**。理由：

1. **不封号** — 企业微信就是为企业和开发者设计的
2. **官方 API** — 有文档、有技术支持、不会突然失效
3. **零扫码** — 配置一次永久生效
4. **更适合仓储场景** — 企业微信天然就是给企业用的，仓库人员都在上面

### Hermes Agent 自己也面临同样问题

注意 Hermes 官方的 WeChat 接入（HermesClaw）只是**社区插件**，不是官方核心功能。它走的就是 wxauto 的路线（Windows + UI Automation）。如果你在 Hermes 的 Discord 社区问微信接入的稳定性，大概率得到的答复也是："it works, but use at your own risk"（能用，但风险自负）。
