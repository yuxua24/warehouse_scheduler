# P0-1：消息通道接入微信 — 详细改造分析

> 分析如何将智能仓储机器人调度系统接入微信（企业微信），让仓库操作人员可以通过微信发送调度指令并接收规划结果。
> 不修改任何代码，仅做技术分析和方案设计。

---

## 一、微信接入方式对比

微信生态中有多种接入方式，适合做 Bot 交互的有以下四种：

| 方式 | 双向通信 | 配置难度 | 公网要求 | 用户侧操作 | 适合场景 |
|------|---------|---------|---------|-----------|---------|
| **企业微信自建应用** | ✅ | 中等 | 需要 | 在企业微信里搜应用名发消息 | 🏆 首选推荐 |
| **企业微信群机器人** | ❌ 仅单向 | 极低 | 不需要 | 只能接收通知，不能发指令 | 纯通知推送 |
| **微信公众号** | ✅（48h内） | 高 | 需要 | 关注公众号后对话 | 面向外部用户 |
| **个人微信**（itchat 等） | ✅ | 低 | 不需要 | 给 Bot 微信号发消息 | ❌ 违反 ToS，会封号 |

### 结论

**推荐使用「企业微信自建应用」**：

- 企业微信是腾讯官方面向企业场景的产品，合规且稳定
- 自建应用支持完整的收发消息能力（文本、Markdown、卡片消息）
- 仓库场景天然适合企业微信（公司内部通讯工具）
- 不需要企业认证也可以创建（注册企业微信即可，个人也能注册）

### 2.3 消息方向

| 方向 | 触发方 | 协议 | 需要什么 |
|------|-------|------|---------|
| **接收**（用户→系统） | 企业微信 | HTTP POST 到我们的回调 URL | 公网可访问的 URL + 加解密密钥 |
| **发送**（系统→用户） | 我们的服务器 | HTTPS GET/POST 到企业微信 API | corp_id + corp_secret → access_token |
| **URL 验证**（首次配置） | 企业微信 | GET 到回调 URL + 参数校验 | 响应 echostr |

#### 3.4.1 `base.py` — Channel 抽象基类

```python
from abc import ABC, abstractmethod
from typing import Optional

class Channel(ABC):
    """消息通道抽象基类"""
    
    name: str  # "wecom", "telegram", etc.
    
    @abstractmethod
    async def send_message(self, user_id: str, content: str) -> bool:
        """向指定用户发送消息，返回是否成功"""
        ...
    
    @abstractmethod
    async def handle_message(self, user_id: str, text: str) -> str:
        """处理用户消息，返回回复内容（调用 workflow 等）"""
        ...
    
    @abstractmethod
    async def verify_url(self, params: dict) -> Optional[str]:
        """首次配置时的 URL 验证，返回 echostr 或 None"""
        ...
    
    @abstractmethod
    async def decrypt_message(self, raw_body: bytes, params: dict) -> dict:
        """解密企业微信 POST 的加密消息体，返回解析后的字典"""
        ...
    
    @abstractmethod
    async def initialize(self):
        """初始化：获取 access_token、加载配置等"""
        ...
```

#### 3.4.2 `crypto.py` — 消息加解密

企业微信使用 **AES-256-CBC** 加密消息体，同时附 **SHA1 签名** 校验。

```python
import base64
import hashlib
import struct
import socket
from Crypto.Cipher import AES

class WeComCrypto:
    """企业微信消息加解密
    
    协议文档: https://developer.work.weixin.qq.com/document/path/90968
    """
    
    def __init__(self, token: str, encoding_aes_key: str, corp_id: str):
        self.token = token
        self.aes_key = base64.b64decode(encoding_aes_key + "=")
        self.corp_id = corp_id
    
    def verify_signature(self, signature: str, timestamp: str, 
                         nonce: str, echostr: str) -> bool:
        """验证 URL 回调签名"""
        sort_list = sorted([self.token, timestamp, nonce, echostr])
        sha1 = hashlib.sha1("".join(sort_list).encode()).hexdigest()
        return sha1 == signature
    
    def decrypt(self, encrypted_msg: str) -> tuple[str, str]:
        """解密消息，返回 (明文, 接收方 corp_id)"""
        # AES-CBC 解密
        cipher = AES.new(self.aes_key, AES.MODE_CBC, self.aes_key[:16])
        plain_text = cipher.decrypt(base64.b64decode(encrypted_msg))
        
        # 去除 PKCS#7 padding
        pad = plain_text[-1]
        content = plain_text[16:-pad]  # 前 16 字节是随机字符串
        
        # 解析: 4字节msg_len + msg + corp_id
        msg_len = socket.ntohl(struct.unpack("I", content[:4])[0])
        msg = content[4:4+msg_len].decode("utf-8")
        receiver_id = content[4+msg_len:].decode("utf-8")
        
        return msg, receiver_id
    
    def encrypt(self, msg: str) -> str:
        """加密回复消息"""
        # 16 字节随机字符串 + 4 字节长度 + 消息 + corp_id
        import secrets
        random_bytes = secrets.token_bytes(16)
        msg_bytes = msg.encode("utf-8")
        msg_len = struct.pack("!I", len(msg_bytes))
        corp_id_bytes = self.corp_id.encode("utf-8")
        
        raw = random_bytes + msg_len + msg_bytes + corp_id_bytes
        
        # PKCS#7 padding
        block_size = 32
        pad = block_size - len(raw) % block_size
        raw += bytes([pad] * pad)
        
        cipher = AES.new(self.aes_key, AES.MODE_CBC, self.aes_key[:16])
        encrypted = cipher.encrypt(raw)
        return base64.b64encode(encrypted).decode()
```

#### 3.4.3 `api_client.py` — 企业微信 API 客户端

```python
import httpx
import time
from typing import Optional

class WeComAPIClient:
    """企业微信 API 客户端
    
    API 文档: https://developer.work.weixin.qq.com/document/path/90664
    """
    
    BASE_URL = "https://qyapi.weixin.qq.com/cgi-bin"
    
    def __init__(self, corp_id: str, corp_secret: str, agent_id: int):
        self.corp_id = corp_id
        self.corp_secret = corp_secret
        self.agent_id = agent_id
        self._access_token: Optional[str] = None
        self._token_expires_at: float = 0
        self._client = httpx.AsyncClient(timeout=10.0)
    
    async def _get_access_token(self) -> str:
        """获取/刷新 access_token（有效期 7200 秒）"""
        if self._access_token and time.time() < self._token_expires_at - 300:
            return self._access_token
        
        resp = await self._client.get(
            f"{self.BASE_URL}/gettoken",
            params={
                "corpid": self.corp_id,
                "corpsecret": self.corp_secret,
            }
        )
        data = resp.json()
        if data.get("errcode") != 0:
            raise Exception(f"获取 token 失败: {data.get('errmsg')}")
        
        self._access_token = data["access_token"]
        self._token_expires_at = time.time() + data.get("expires_in", 7200)
        return self._access_token
    
    async def send_text_message(self, user_id: str, content: str) -> dict:
        """发送文本消息到指定用户"""
        token = await self._get_access_token()
        resp = await self._client.post(
            f"{self.BASE_URL}/message/send",
            params={"access_token": token},
            json={
                "touser": user_id,
                "msgtype": "text",
                "agentid": self.agent_id,
                "text": {"content": content},
            }
        )
        return resp.json()
    
    async def send_markdown_message(self, user_id: str, content: str) -> dict:
        """发送 Markdown 消息（企业微信支持部分 Markdown 语法）"""
        token = await self._get_access_token()
        resp = await self._client.post(
            f"{self.BASE_URL}/message/send",
            params={"access_token": token},
            json={
                "touser": user_id,
                "msgtype": "markdown",
                "agentid": self.agent_id,
                "markdown": {"content": content},
            }
        )
        return resp.json()
```

#### 3.4.4 `message_parser.py` — 消息解析

```python
import xml.etree.ElementTree as ET

def parse_wecom_xml(xml_text: str) -> dict:
    """解析企业微信回调的 XML 消息体
    
    示例 XML:
    <xml>
        <ToUserName><![CDATA[corp_id]]></ToUserName>
        <FromUserName><![CDATA[UserID]]></FromUserName>
        <CreateTime>1408091189</CreateTime>
        <MsgType><![CDATA[text]]></MsgType>
        <Content><![CDATA[R1前往装卸区]]></Content>
        <MsgId>1234567890123456</MsgId>
        <AgentID>1000002</AgentID>
    </xml>
    """
    root = ET.fromstring(xml_text)
    
    def _get(name: str) -> str:
        elem = root.find(name)
        return elem.text if elem is not None else ""
    
    return {
        "to_user_name": _get("ToUserName"),
        "from_user_name": _get("FromUserName"),
        "create_time": int(_get("CreateTime") or 0),
        "msg_type": _get("MsgType"),
        "content": _get("Content"),
        "msg_id": _get("MsgId"),
        "agent_id": _get("AgentID"),
    }
```

#### 3.4.5 `handler.py` — 主处理器

```python
from app.channels.wecom.crypto import WeComCrypto
from app.channels.wecom.api_client import WeComAPIClient
from app.channels.wecom.message_parser import parse_wecom_xml
from app.orchestration.workflow import Workflow

class WeComHandler:
    """企业微信消息处理器"""
    
    def __init__(self, config: dict, workflow: Workflow):
        self.crypto = WeComCrypto(
            token=config["token"],
            encoding_aes_key=config["encoding_aes_key"],
            corp_id=config["corp_id"],
        )
        self.api = WeComAPIClient(
            corp_id=config["corp_id"],
            corp_secret=config["corp_secret"],
            agent_id=config["agent_id"],
        )
        self.workflow = workflow
        self.allowed_users = set(config.get("allowed_users", []))
    
    async def verify_url(self, msg_signature: str, timestamp: str,
                         nonce: str, echostr: str) -> str:
        """处理首次配置时的 URL 验证"""
        if self.crypto.verify_signature(msg_signature, timestamp, nonce, echostr):
            decrypted, _ = self.crypto.decrypt(echostr)
            return decrypted
        raise ValueError("签名验证失败")
    
    async def handle_callback(self, msg_signature: str, timestamp: str,
                              nonce: str, raw_body: bytes) -> str:
        """处理企业微信 POST 的回调消息"""
        # 1. 解密消息体
        xml_text, _ = self.crypto.decrypt(raw_body.decode())
        
        # 2. 解析 XML
        msg = parse_wecom_xml(xml_text)
        
        # 3. 鉴权：仅允许授权用户
        if msg["from_user_name"] not in self.allowed_users:
            return self._build_text_reply(
                msg, "❌ 您没有权限使用此功能，请联系管理员"
            )
        
        # 4. 根据消息类型处理
        if msg["msg_type"] == "text":
            return await self._handle_text_message(msg)
        elif msg["msg_type"] == "event":
            return await self._handle_event(msg)
        else:
            return self._build_text_reply(
                msg, "⚠️ 目前仅支持文本指令，请用自然语言描述调度任务"
            )
    
    async def _handle_text_message(self, msg: dict) -> str:
        """处理文本消息：作为调度指令传给 Workflow"""
        instruction = msg["content"].strip()
        
        if not instruction:
            return self._build_text_reply(msg, "请输入调度指令")
        
        # 帮助命令
        if instruction in ("帮助", "help", "?"):
            return self._build_text_reply(msg, HELP_TEXT)
        
        try:
            # 调用调度引擎
            state = self.workflow.run(instruction)
            
            # 格式化结果
            reply = format_schedule_result(state)
            return self._build_markdown_reply(msg, reply)
        except Exception as e:
            return self._build_text_reply(msg, f"❌ 调度失败: {str(e)}")
    
    def _build_text_reply(self, msg: dict, content: str) -> str:
        """构建文本回复 XML"""
        return f"""<xml>
<ToUserName><![CDATA[{msg["from_user_name"]}]]></ToUserName>
<FromUserName><![CDATA[{msg["to_user_name"]}]]></FromUserName>
<CreateTime>{int(time.time())}</CreateTime>
<MsgType><![CDATA[text]]></MsgType>
<Content><![CDATA[{content}]]></Content>
</xml>"""
    
    def _build_markdown_reply(self, msg: dict, content: str) -> str:
        """构建 Markdown 回复 XML"""
        return f"""<xml>
<ToUserName><![CDATA[{msg["from_user_name"]}]]></ToUserName>
<FromUserName><![CDATA[{msg["to_user_name"]}]]></FromUserName>
<CreateTime>{int(time.time())}</CreateTime>
<MsgType><![CDATA[markdown]]></MsgType>
<Markdown><Content><![CDATA[{content}]]></Content></Markdown>
</xml>"""


def format_schedule_result(state) -> str:
    """将 PlanningState 格式化为易读的回复文本"""
    status_emoji = {
        "succeeded": "✅",
        "partially_succeeded": "⚠️",
        "infeasible": "❌",
    }
    emoji = status_emoji.get(state.status.value, "❓")
    
    lines = [f"{emoji} 调度结果: {state.status.value}"]
    lines.append("━━━━━━━━━━━━━━")
    for tr in state.task_results:
        icon = "✅" if tr.success else "❌"
        steps = len(tr.path)
        lines.append(f"{icon} **{tr.robot_id}**: {steps} 步"
                     f"{' | 失败: ' + tr.failure_reason if not tr.success else ''}")
    
    lines.append("━━━━━━━━━━━━━━")
    if state.metrics:
        lines.append(f"成功率: **{state.metrics.planning_success_rate:.0%}**"
                     f" | 耗时: {state.metrics.total_planning_time_ms:.0f}ms")
        lines.append(f"冲突: {state.metrics.final_conflict_count} 次"
                     f" | 重规划: {state.metrics.retry_count} 次")
    
    return "\n".join(lines)
```

#### 3.4.6 `router.py` — FastAPI 路由

```python
from fastapi import APIRouter, Request, Query, HTTPException
from app.channels.wecom.handler import WeComHandler

wecom_router = APIRouter(prefix="/api/wecom", tags=["wecom"])

# handler 在 server.py 启动时初始化并挂载
_wecom_handler: WeComHandler = None

def init_wecom(handler: WeComHandler):
    global _wecom_handler
    _wecom_handler = handler

@wecom_router.get("/callback")
async def wecom_verify(
    msg_signature: str = Query(...),
    timestamp: str = Query(...),
    nonce: str = Query(...),
    echostr: str = Query(...),
):
    """企业微信首次配置时的 URL 验证"""
    try:
        decrypted = await _wecom_handler.verify_url(
            msg_signature, timestamp, nonce, echostr
        )
        return PlainTextResponse(content=decrypted)
    except Exception as e:
        raise HTTPException(status_code=403, detail=str(e))

@wecom_router.post("/callback")
async def wecom_callback(
    request: Request,
    msg_signature: str = Query(...),
    timestamp: str = Query(...),
    nonce: str = Query(...),
):
    """接收企业微信推送的消息"""
    raw_body = await request.body()
    try:
        reply_xml = await _wecom_handler.handle_callback(
            msg_signature, timestamp, nonce, raw_body
        )
        return PlainTextResponse(content=reply_xml, media_type="application/xml")
    except Exception as e:
        # 出错时返回 success（避免企业微信重复推送）
        return PlainTextResponse(content="success")
```

## 四、安全性设计

### 4.1 多层防护

```
┌─────────────────────────────────────────────────────┐
│ 第一层: 企业微信服务器 IP 白名单                        │
│   - 仅接受来自企业微信官方 IP 段的请求                   │
│   - 可以配置在 Nginx/防火墙层                           │
├─────────────────────────────────────────────────────┤
│ 第二层: SHA1 签名校验                                 │
│   - 每条消息附带 msg_signature                         │
│   - 用 token + timestamp + nonce 重新计算签名比对        │
├─────────────────────────────────────────────────────┤
│ 第三层: 用户白名单                                    │
│   - allowed_users 列表                                 │
│   - 非白名单用户的消息直接返回权限拒绝                     │
├─────────────────────────────────────────────────────┤
│ 第四层: 内容限制                                      │
│   - 消息长度限制（≤ 2000 字符）                         │
│   - 拒绝非文本消息类型                                  │
└─────────────────────────────────────────────────────┘
```

### 4.2 敏感信息保护

- `wecom_config.json` 加入 `.gitignore`，**不提交到 Git**
- `corp_secret` 和 `encoding_aes_key` 是核心秘钥，泄露后需立即在企业微信后台重置
- 日志中不得记录完整的消息体和 access_token

## 五、部署注意事项

### 5.1 公网回调 URL

企业微信的**回调 URL 必须是可以从公网访问的 HTTPS URL**。

**方案 A：直接部署到有公网 IP 的服务器**
- 最稳定
- 需要一台云服务器（阿里云/腾讯云/AWS 入门级即可）

**方案 B：开发阶段使用 ngrok 隧道**
```bash
# 将本地 8000 端口暴露到公网
ngrok http 8000

# 得到 https://abc123.ngrok.io
# 配置回调 URL: https://abc123.ngrok.io/api/wecom/callback
```
- 适合开发调试
- 每次重启 ngrok URL 会变化，需要重新配置

**方案 C：使用 Cloudflare Tunnel**（免费、稳定、URL 不变）
```bash
cloudflared tunnel --url http://localhost:8000
```

### 5.2 access_token 刷新

企业微信 access_token 有效期 7200 秒（2 小时），需要在 `api_client.py` 中自动管理刷新：

```python
# 伪代码逻辑
if token is None or time.time() > expires_at - 300:  # 提前 5 分钟刷新
    token = await fetch_new_token()
```

### 5.3 消息去重

企业微信可能因网络重试**重复推送同一条消息**。通过 msg_id 去重：

```python
# WeComHandler 中维护一个最近 msg_id 集合
_recent_msg_ids: set = set()  # 容量 1000，LRU 淘汰

if msg["msg_id"] in self._recent_msg_ids:
    return "success"  # 已处理，返回成功避免重复推送
self._recent_msg_ids.add(msg["msg_id"])
```
